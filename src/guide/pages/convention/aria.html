1. 지원되는 상태 및 속성

WAI-ARIA(Web Accessibility Initiative - Accessible Rich Internet Applications)는 웹 콘텐츠와 웹 애플리케이션을 보다 접근성 있게 만들기 위한 기술 명세입니다. 이 가이드는 WAI-ARIA 1.2의 최신 권고 사항을 기반으로, 웹 개발 시 사용하는 주요 속성 및 상태에 대해 분류하고 설명합니다.

1-1. 번역 가능한 상태와 속성

HTML 사양은 다른 사양들이 번역 가능한 속성을 정의할 수 있음을 명시합니다. 보조 기술 사용자들에게 이해될 수 있도록, 다음 상태와 속성의 값은 번역 가능한 속성이며 페이지가 현지화될 때 번역되어야 합니다:

  • aria-label
  • aria-placeholder
  • aria-roledescription
  • aria-valuetext

1-2. 전역 상태와 속성

일부 상태와 속성은 역할이 적용되었는지 여부와 상관없이 모든 호스트 언어 요소에 적용됩니다. 다음 전역 상태와 속성은 모든 역할과 모든 기본 마크업 요소에서 지원되며, 달리 금지되지 않는 한 사용할 수 있습니다. 역할이 어떤 전역 상태나 속성의 사용을 금지하는 경우, 그 상태나 속성은 역할을 정의하는 섹션에 포함된 특성 표에서 금지된 것으로 나열됩니다.

  • aria-atomic
  • aria-busy (state)
  • aria-controls
  • aria-current (state)
  • aria-describedby
  • aria-details
  • aria-disabled (state) (Global use deprecated in ARIA 1.2)
  • aria-dropeffect
  • aria-errormessage (Global use deprecated in ARIA 1.2)
  • aria-flowto
  • aria-grabbed (state)
  • aria-haspopup (Global use deprecated in ARIA 1.2)
  • aria-hidden (state)
  • aria-invalid (state) (Global use deprecated in ARIA 1.2)
  • aria-keyshortcuts
  • aria-label (Except where prohibited)
  • aria-labelledby (Except where prohibited)
  • aria-live
  • aria-owns
  • aria-relevant
  • aria-roledescription

1-3. WAI-ARIA 상태 및 속성 분류

상태와 속성은 다음과 같이 분류됩니다.

  • Widget Attributes (위젯 속성): 사용자 인터페이스 구성 요소의 상태, 속성 및 기능을 정의합니다.
  • Live Region Attributes (라이브 영역 속성): 동적으로 변경되는 콘텐츠의 업데이트 방식을 보조 기술에 알립니다.
  • Drag-and-Drop Attributes (드래그 앤 드롭 속성): 드래그 앤 드롭 인터랙션을 지원하는 요소의 속성을 정의합니다.
  • Relationship Attributes (관계 속성): 문서 내 요소 간의 관계를 정의합니다.

1-3-1. Widget Attributes

이 섹션은 GUI 시스템이나 풍부한 인터넷 응용 프로그램에서 사용자 입력을 받고 사용자 동작을 처리하는 일반적인 사용자 인터페이스 요소에 특화된 속성들을 포함합니다. 이러한 속성들은 위젯 역할을 지원하기 위해 사용됩니다.

  • aria-autocomplete
  • aria-checked
  • aria-disabled
  • aria-errormessage
  • aria-expanded
  • aria-haspopup
  • aria-hidden
  • aria-invalid
  • aria-label
  • aria-level
  • aria-modal
  • aria-multiline
  • aria-multiselectable
  • aria-orientation
  • aria-placeholder
  • aria-pressed
  • aria-readonly
  • aria-required
  • aria-selected
  • aria-sort
  • aria-valuemax
  • aria-valuemin
  • aria-valuenow
  • aria-valuetext

위젯 속성은 사용자 에이전트에 의해 플랫폼 접근성 API 상태로 매핑될 수 있으며, 이는 보조 기술에 의해 접근될 수 있습니다. 또는, DOM에서 직접 접근될 수도 있습니다. 사용자 에이전트는 상태가 변경될 때 보조 기술이 알림을 받을 수 있는 방법을 제공해야 합니다(반드시 제공해야 합니다). 이는 DOM 속성 변경 이벤트나 플랫폼 접근성 API 이벤트를 통해 가능합니다.

1-3-2. Live Region Attributes

이 섹션은 풍부한 인터넷 응용 프로그램에서 실시간 영역에 특화된 속성들을 포함합니다. 이러한 속성은 어떤 요소에도 적용될 수 있습니다. 이 속성들의 목적은 요소가 포커스를 가지고 있지 않을 때도 내용 변경이 발생할 수 있음을 나타내고, 그러한 내용 업데이트를 처리하는 방법에 대한 정보를 보조 기술에 제공하는 것입니다. 일부 역할은 해당 역할에 특정한 aria-live 속성의 기본값을 지정합니다. 실시간 영역의 예로는 업데이트되는 주식 시세를 나열하는 티커 섹션이 있습니다.

  • aria-atomic
  • aria-busy
  • aria-live
  • aria-relevant

1-3-3. Drag-and-Drop Attributes

이 섹션은 드래그 앤 드롭 인터페이스 요소에 대한 정보를 나타내는 속성을 나열합니다. 여기에는 드래그 가능한 요소와 그들의 드롭 대상 등이 포함됩니다. 드롭 대상 정보는 저자에 의해 시각적으로 렌더링되고, 대체 모달리티를 통해 보조 기술에 제공될 것입니다.

  • aria-dropeffect
  • aria-grabbed

1-3-4. Relationship Attributes

이 섹션은 문서 구조에서 쉽게 파악할 수 없는 요소들 사이의 관계나 연관성을 나타내는 속성들을 나열합니다.

  • aria-activedescendant
  • aria-colcount
  • aria-colindex
  • aria-colspan
  • aria-controls
  • aria-describedby
  • aria-details
  • aria-errormessage
  • aria-flowto
  • aria-labelledby
  • aria-owns
  • aria-posinset
  • aria-rowcount
  • aria-rowindex
  • aria-rowspan
  • aria-setsize

1-4. 상태 및 속성 정의(모든 aria-* 속성) - 48개

스크린리더 지원여부 확인 사이트

1-4-1. aria-activedescendant (Property) - undefined (기본값)

  • 용도: 컴포넌트 내에서 현재 활성화된 항목을 가리키는데 사용됩니다. 컨테이너 또는 복합 위젯 내에서 키보드 탐색에 사용됩니다.
  • 적용 컴포넌트: 콤보박스(Combobox), 리스트박스(Listbox), 메뉴(Menu), 트리뷰(Treeview)
  • 스크린리더 지원여부: JAWS (supported), NVDA (supported), VoiceOver (iOS) (supported), TalkBack (supported), Narrator (supported)

1-4-2. aria-atomic (Property) - false (기본값) | true

  • 용도: 라이브 영역에 대한 변경 사항이 전체적으로 또는 부분적으로 스크린 리더에 의해 발표되어야 하는지 여부를 나타냅니다.
  • 적용 컴포넌트: 알림(Alert), 상태 표시줄(Status Bar)
  • 스크린리더 지원여부: JAWS (partial), NVDA (supported), VoiceOver (iOS) (supported), TalkBack (supported), Narrator (partial)

1-4-3. aria-autocomplete (Property) - none (기본값) | inline | list | both

  • 용도: 사용자가 입력을 시작할 때 입력값을 자동 완성할 수 있는지 여부를 나타냅니다.
  • 적용 컴포넌트: 입력 필드(Input Field), 콤보박스(Combobox)
  • 스크린리더 지원여부: JAWS (none), NVDA (some partial support), VoiceOver (iOS) (none), TalkBack (none), Narrator (none)

1-4-4. aria-busy (State) - false (기본값) | true

  • 용도: 요소가 현재 업데이트 중임을 나타내어 사용자 인터페이스의 나머지 부분이 현재 사용할 수 없거나 바쁘다는 것을 사용자에게 알립니다.
  • 적용 컴포넌트: 페이지 또는 섹션 로딩 중(Loading Spinners), 데이터 그리드(Data Grids)
  • 스크린리더 지원여부: JAWS (supported), NVDA (supported), VoiceOver (iOS) (none), TalkBack (none), Narrator (supported)

1-4-5. aria-checked (State) - false (기본값) | true | mixed | undefined

  • 용도: 체크박스, 라디오 버튼, 또는 스위치의 체크 상태를 나타냅니다.
  • 적용 컴포넌트: 체크박스(Checkbox), 라디오 버튼(Radio Button), 스위치(Switch)
  • 스크린리더 지원여부: JAWS (supported), NVDA (supported), VoiceOver (iOS) (supported), TalkBack (supported), Narrator (supported)

1-4-6. aria-colcount (Property) - undefined (기본값)

  • 용도: 그리드나 테이블에 있는 열의 총 수를 나타냅니다.
  • 적용 컴포넌트: 그리드(Grid), 테이블(Table)
  • 스크린리더 지원여부: JAWS (partial), NVDA (unknown), VoiceOver (iOS) (none), TalkBack (unknown), Narrator (partial)

1-4-7. aria-colindex (Property) - undefined (기본값)

  • 용도: 그리드나 테이블 내에서 셀의 열 위치를 나타냅니다.
  • 적용 컴포넌트: 그리드 셀(Grid Cell), 테이블 셀(Table Cell)
  • 스크린리더 지원여부: JAWS (unknown), NVDA (unknown), VoiceOver (iOS) (unknown), TalkBack (unknown), Narrator (unknown)

1-4-8. aria-colspan (Property) - undefined (기본값)

  • 용도: 셀이 차지하는 열의 수를 나타냅니다. 테이블이나 그리드 구조에서 사용됩니다.
  • 적용 컴포넌트: 그리드 셀(Grid Cell), 테이블 셀(Table Cell)
  • 스크린리더 지원여부: JAWS (unknown), NVDA (unknown), VoiceOver (iOS) (unknown), TalkBack (unknown), Narrator (unknown)

1-4-9. aria-controls (Property) - undefined (기본값)

  • 용도: 현재 요소가 제어하는 다른 요소의 ID를 참조합니다. 주로 탭, 드롭다운 메뉴, 커스텀 슬라이더 등에서 사용됩니다.
  • 적용 컴포넌트: 탭(Tab), 드롭다운 메뉴(Dropdown Menu), 슬라이더(Slider)
  • 스크린리더 지원여부: JAWS (supported), NVDA (supported), VoiceOver (iOS) (none), TalkBack (none), Narrator (supported)

1-4-10. aria-current (State) - false (기본값) | true | page | step | location | date | time

  • 용도: 현재 요소가 페이지, 단계, 위치 등에 대한 현재 상태를 나타냅니다. 링크, 메뉴 항목, 또는 스텝 인디케이터에 사용됩니다.
  • 적용 컴포넌트: 링크(Link), 메뉴 항목(Menu Item), 스텝 인디케이터(Step Indicator)
  • 스크린리더 지원여부: JAWS (supported), NVDA (supported), VoiceOver (iOS) (supported), TalkBack (supported), Narrator (supported)

1-4-11. aria-describedby (Property) - undefined (기본값)

  • 용도: 현재 요소에 대한 추가 설명이나 설명을 제공하는 요소의 ID를 참조합니다. 복잡한 위젯, 이미지, 또는 인터랙션에 대한 설명 제공 시 사용됩니다.
  • 적용 컴포넌트: 위젯(Widget), 이미지(Image), 버튼(Button)
  • 스크린리더 지원여부: JAWS (supported), NVDA (supported), VoiceOver (iOS) (supported), TalkBack (supported), Narrator (supported)

1-4-12. aria-details (Property) - undefined (기본값)

  • 용도: 현재 요소의 복잡한 정보에 대한 상세 정보를 제공하는 요소의 ID를 참조합니다. 주로 문서, 이미지, 또는 다른 요소들에 대한 상세 설명을 제공하는 데 사용됩니다.
  • 적용 컴포넌트: 문서(Document), 이미지(Image), 그래프(Graph)
  • 스크린리더 지원여부: JAWS (partial), NVDA (partial), VoiceOver (iOS) (partial), TalkBack (unknown), Narrator (partial)

1-4-13. aria-disabled (State) - false (기본값) | true

  • 용도: 요소가 비활성화되었음을 나타냅니다. 비활성화된 요소는 사용자 인터랙션을 받지 않습니다.
  • 적용 컴포넌트: 버튼(Button), 링크(Link), 입력 필드(Input Field)
  • 스크린리더 지원여부: JAWS (supported), NVDA (supported), VoiceOver (iOS) (supported), TalkBack (supported), Narrator (supported)

1-4-14. aria-dropeffect (Property) - none (기본값) | copy | execute | link | move | popup

  • 용도: 드래그 앤 드롭 작업 중 객체에 적용될 수 있는 효과를 나타냅니다.
  • 적용 컴포넌트: 드래그 가능한 아이템(Draggable Item)
  • 스크린리더 지원여부: 현재 대부분의 스크린리더와 브라우저에서는 이 속성을 지원하지 않습니다.

1-5. 용어 설명 - IDREF

IDREF는 HTML에서 사용되는 용어로, "ID Reference"의 약자입니다. 이는 특정 HTML 요소의 id 속성 값에 대한 참조를 의미합니다. 즉, 어떤 속성의 값으로 IDREF를 사용한다는 것은, 해당 속성이 다른 요소의 id 값을 참조하여 그 요소를 식별하게 됨을 의미합니다.

예를 들어, aria-activedescendant 속성의 경우, 이 속성이 설정된 요소는 키보드 네비게이션을 통해 현재 활성화 또는 선택된 상태를 가지는 자식 요소의 id 값을 참조합니다. 이를 통해 사용자가 키보드로 네비게이션할 때, 활성화된 요소를 보조 기술을 통해 식별하고 상호작용할 수 있게 됩니다.

<div role="listbox" aria-activedescendant="listitem3">
  <div id="listitem1" role="option">Item 1</div>
  <div id="listitem2" role="option">Item 2</div>
  <div id="listitem3" role="option">Item 3</div>
  <div id="listitem4" role="option">Item 4</div>
</div>

위 예제에서, div 요소는 role="listbox"를 가지며, aria-activedescendant 속성을 사용하여 현재 활성화된 항목을 id="listitem3"으로 지정합니다. 따라서 "Item 3"이 현재 선택되어 있는 상태임을 나타냅니다. IDREF를 사용함으로써, 해당 listbox는 "Item 3"이 활성 항목임을 보조 기술에 명확하게 알릴 수 있습니다.
