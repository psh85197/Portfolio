1. 역할 모델

이 섹션에서는 WAI-ARIA 역할을 정의하고 그 특성과 속성에 대해 설명합니다.

역할, 그 특성, 그들이 지원하는 상태와 속성, 그리고 마크업에서의 사용 방법에 대한 명세는 규범으로 간주해야 합니다.

DOM에서의 내용을 반영하기 위해, 사용자 에이전트는 역할 속성을 구현된 접근성 API에서 적절한 값으로 매핑해야 합니다(SHOULD), 그리고 역할 속성이 변경될 때 매핑을 업데이트해야 합니다(SHOULD).

1-1. 역할의 분류

현재 사용자 시나리오를 지원하기 위해, 이 사양은 사용자 인터페이스 위젯(슬라이더, 트리 컨트롤 등)을 정의하는 역할과 페이지 구조(섹션, 네비게이션 등)를 정의하는 역할로 역할들을 분류합니다. application 또는 document 역할로 표시된 영역에 대해 일부 보조 기술은 특별한 상호 작용 모드를 제공한다는 점에 주목하세요.

역할 간의 관계를 시각적으로 설명하는 것은 ARIA 1.2 클래스 다이어그램에서 사용할 수 있습니다.

역할은 다음과 같이 분류됩니다:

  1. Abstract Roles
  1. Widget Roles
  1. Document Structure Roles
  1. Landmark Roles
  1. Live Region Roles
  1. Window Roles

1-1-1. Abstract Roles

다음 역할들은 일반적인 역할 개념을 정의하기 위한 목적으로 WAI-ARIA 역할 모델을 지원하기 위해 사용됩니다.

추상 역할은 온톨로지를 위해 사용됩니다. 작성자는 내용에서 추상 역할을 절대 사용해서는 안 됩니다.

  • sectionhead
  • select

  • widget
  • window

1-2. 역할의 정의

1-2-1. 역할(Role):

역할은 HTML 태그에 추가적인 의미를 부여합니다. 예를 들어, 'button' 역할은 요소가 버튼으로 작동함을 나타냅니다. 이는 보조 기술이 해당 요소를 사용자에게 버튼으로서 설명할 수 있게 합니다.

1-2-2. 대표 역할 목록:

  • landmark roles: 페이지 내 주요 영역을 나타냅니다. ('banner', 'navigation', 'main', 'complementary', 'contentinfo')
  • widget roles: 사용자 인터페이스 위젯을 나타냅니다. ('button', 'checkbox', 'progressbar', 'slider')
  • document structure roles: 문서 구조를 나타냅니다. ('article', 'list', 'listitem')

1-2-3. 사용상의 주의점:

  • 가능하면 네이티브 HTML 요소를 사용하세요. 네이티브 요소는 기본적으로 접근성이 좋습니다.
  • WAI-ARIA를 사용할 때는 역할, 속성, 상태를 정확히 사용해야 합니다. 잘못된 사용은 오히려 접근성 문제를 일으킬 수 있습니다.

1-3. Characteristics

"Characteristics" 섹션은 특정 WAI-ARIA 역할의 기본적인 특성과 이 역할이 어떻게 사용되어야 하는지에 대한 상세 정보를 제공합니다. 여기에는 역할의 추상 여부, 상위 클래스 역할, 관련 개념, 필요한 상태 및 속성, 상속된 상태 및 속성, 그리고 금지된 상태 및 속성 등이 포함될 수 있습니다. 이 정보는 웹 개발자들이 접근성 있는 웹 콘텐츠를 구현할 때 해당 역할을 올바르게 사용하는 데 도움을 줍니다. 자세한 내용은 WAI-ARIA 문서를 참조하세요.

  • Is Abstract (추상 여부): 해당 역할이 직접 사용될 수 있는지, 아니면 상위 역할로만 사용되어야 하는지 나타냅니다.
  • Superclass Role (상위 클래스 역할): 현재 역할의 상위 역할이 무엇인지 나타냅니다.
  • Base Concept (기본 개념): 해당 역할과 관련된 HTML 기본 요소나 개념입니다.
  • Related Concepts (관련 개념): 다른 기술 또는 명세에서 유사한 개념이나 역할입니다.
  • Required Context Role (필요한 컨텍스트 역할): 해당 역할을 사용하기 위해 필요한 상위 역할입니다.
  • Subclass Roles (하위 클래스 역할): 현재 역할에서 파생될 수 있는 하위 역할입니다.
  • Required Owned Elements (필수 소유 요소): 해당 역할을 가진 요소가 포함해야 하는 자식 요소입니다.
  • Supported States and Properties (지원되는 상태 및 속성): 해당 역할에 사용할 수 있는 ARIA 상태 및 속성입니다.
  • Inherited States and Properties (상속된 상태 및 속성): 상위 역할에서 상속받은 상태 및 속성입니다.
  • Prohibited States and Properties (금지된 상태 및 속성): 해당 역할에 사용해서는 안 되는 ARIA 상태 및 속성입니다.
  • Name From (이름 출처): 요소의 접근 가능한 이름이 어디서 유래되는지 나타냅니다.
  • Implicit Value for Role (역할에 대한 암시적 값): 역할에 자동으로 적용되는 기본값입니다.
  • Accessible Name Required (접근 가능한 이름 필요 여부): 해당 역할이 접근 가능한 이름을 필요로 하는지 여부입니다.
  • Children Presentational (자식 프레젠테이션 여부): 자식 요소가 프레젠테이션(시각적 표현) 목적으로만 사용되는지 나타냅니다.
